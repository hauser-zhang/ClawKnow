# 知识库智能问答

## 元数据

| 项目 | 内容 |
|------|------|
| 用途 | 以本地知识库为首要知识源回答 AI/ML 技术问题，支持联网补充 |
| 输入 | 用户的技术问题（自然语言）|
| 输出 | 基于知识库的结构化回答（含来源标注）|
| 副作用 | **无**（只读，不修改任何文件）|
| 需要确认 | 否 |

## 触发条件

**会触发（AI/ML 技术问题 + 项目上下文）：**
- "MoE 和 Dense Model 有什么区别？"
- "解释一下 GRPO 的训练目标"
- "KV Cache 的内存占用怎么计算？"
- "知识库里有关于 Flash Attention 的内容吗？"
- "帮我搜一下最新的 RLHF 论文"

**不触发（明确委托给其他技能或不相关）：**
- "归档一下" → archive
- "同步到飞书" → sync-wiki
- "帮我规划知识库" → plan-wiki
- "帮我记录一下面试题" → interview
- "这段 Python 代码为什么报错" → 直接回答，无需检索
- "帮我写一个 README" → 不相关

## 执行流程

### 第一步：检索知识库

提取用户问题的核心关键词（1–3 个）：

```bash
# 默认 workspace
python ${CLAUDE_SKILL_DIR}/scripts/search_kb.py "<关键词>"

# 指定 workspace（用户提到某个知识库名称时）
python ${CLAUDE_SKILL_DIR}/scripts/search_kb.py --kb <kb_id> "<关键词>"
```

### 第二步：组织回答

**知识库有匹配内容：**
- 以"根据知识库（路径：XXX > YYY）"开头
- 基于知识库内容给出结构化回答
- 结尾提示"如需更深入内容，可以让我联网搜索"

**知识库无匹配：**
- 说明"知识库中暂无此内容"
- 直接给出基于通用知识的回答
- 建议"是否需要联网查最新资料？"

### 第三步（按需）：联网搜索

用户说"搜一下"、"查一下"、"联网"时，使用 WebSearch：
1. 整合知识库内容 + 搜索结果
2. 明确标注哪些来自知识库，哪些来自搜索
3. 附上搜索来源链接

### 第四步（按需）：提示归档

一轮深度讨论结束后，如产生了新的知识要点，可简短提示：

> "本次讨论产生了一些新内容，如需保存到知识库，直接说'归档'即可。"

**不主动执行归档**——等用户明确说"归档"，由 archive skill 处理。

## 知识库不存在时

如果 `workspaces/<kb_id>/knowledge_tree.json` 不存在：
- 直接用通用知识回答问题
- 提示"当前知识库为空，如需建立知识库请告知"
