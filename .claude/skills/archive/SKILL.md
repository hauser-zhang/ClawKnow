# 归档讨论到知识库

将对话中产生的新知识要点归档到知识树。

## 完整流程

### 第一步：提炼知识要点

回顾当前对话，提取值得保存的内容：
- 核心概念和定义
- 关键结论和对比
- 重要的技术细节
- 联网搜索获得的新信息

**去掉**：闲聊、重复内容、过程性讨论。

### 第二步：定位目标节点

1. 读取 `data/knowledge_tree.json`
2. 判断内容应归入哪个节点
3. 可能的情况：
   - **匹配现有节点**：直接追加到该节点的 summary
   - **需要新子节点**：在合适的父节点下创建新节点
   - **需要新分类**：建议在顶层创建新分类

### 第三步：确认归档方案

向用户展示：
```
归档内容：
  [简要摘要]

目标节点：
  知识库 > 模型架构 > MoE

操作方式：
  更新现有节点的 summary / 创建新子节点
```

等待用户确认或调整。

### 第四步：执行归档

用户确认后运行：

```bash
python ${CLAUDE_SKILL_DIR}/scripts/archive_to_kb.py "<节点路径>" "<归档内容>"
```

路径用 `>` 分隔，如 `"模型架构>MoE"`。

如果需要创建新节点，直接编辑 `data/knowledge_tree.json`。

### 第五步：后续提示

- 告知归档成功
- 如果知识库已同步过飞书（节点有 obj_token），提醒用户可运行 `/sync-wiki` 更新飞书端
- 如果是新增节点，说明该节点尚未同步到飞书

## 归档内容格式

写入 summary 时使用简洁的文本格式：

```
[原有内容]

---
[日期] 讨论补充：
- 要点 1
- 要点 2
详细：...
```
